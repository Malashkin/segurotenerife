# QA-тестирование агента: 30 персон, живой прогон

Реальное e2e-тестирование чат-агента по прод-API (`/api/chat` живым Claude +
`/api/handoff`). Цель: оценить качество ответов и конверсию в лиды, найти что
улучшить.

## Методология

- **Harness:** `scripts/qa-personas.mjs` (29 персон) + `scripts/qa-supplemental.mjs`
  (паритет 4 языков + пробелы по потребностям). Multi-turn с реальной историей,
  дросселирование под rate limit, ретрай 429.
- **Покрытие:** 4 языка (ru/uk/en/es, сбалансированно) × все 9 потребностей
  корпуса (med, family, student, reembolso, travel, dental, life, decesos, pet)
  + профили: ВНЖ, студент, семья, пенсионер, цифровой кочевник, автономо,
  беременность, хронические болезни, питомец, скептик-провокатор и др.
- **Объём:** ~110 живых вызовов Claude, до 5 вопросов на персону, + лид на каждую.

## Сильные стороны (подтверждены на реальных ответах)

- **Бренд-нейтральность железная** — даже на прямой провокации («с какой
  компанией работаете?», «почему не называете бренд?») агент корректно уходит:
  «работаем с несколькими партнёрами… конкретного страховщика назовёт менеджер».
- **Исчерпывающие, точные ответы** — сертификат для консульства, без копэй и
  периодов ожидания, список покрытия, лимиты возмещения; один уточняющий вопрос,
  когда вид страховки неясен.
- **Тёплый, заботливый тон** (после доработки) — «понимаю вас, переезд с детьми
  это волнительно…».

## Найденные проблемы → исправлено

1. **Тип страховки в лиде схлопывался в «med».** Причина: backend брал первый
   интент дока, фронт перезаписывал тему. → Тема = ПЕРВАЯ потребность (или интент
   карточки), не перезаписывается; backend эхо-возвращает интент. ✅
2. **Хендофф пропускал явные запросы** — укр. «з'єднайте» (апостроф U+2019) и
   ценовые вопросы. → `wants_manager`: нормализация апострофов + группа
   PRICE/BUY (цена/расчёт/оформление → к менеджеру). ✅ Проверено: Q4 «цена» на
   всех 4 языках → handoff.
3. **Формальное закрытие / кривые фразы** («выбор провайдера, оформление
   сертификата»). → Промпт: закрытие = простое приглашение к бесплатному расчёту
   (менеджер = расчёт стоимости). ✅
4. **Сухой/формальный тон** → тёплый, человечный, с заботой. ✅
5. **Утечка испанских слов** (urgencias, nómada digital) в ответ на другом
   языке → инструкция переводить. ✅
6. **Кнопка «к менеджеру» после первого вопроса** (рано) → после ≥2 вопросов;
   на сигнал агента — инлайн-кнопка, авто-карточка только по 60с бездействия. ✅
7. **Пробелы аналитики** → добавлены `answer_received`, `agent_fallback`,
   `lead_submitted` (конверсия), `tg_message_copied`, `lang_switched`. ✅

## Остаточные наблюдения (низкий приоритет)

- **Markdown `**`** в ~40% ответов — фронт срезает (`cleanReply`), пользователь
  не видит. Можно усилить промпт, но визуально проблемы нет.
- **Редкие опечатки Haiku** («коверкум», «прортезы») и единичный мис-ретривал
  (вопрос «оставить своего врача?» поднял студенческий док). Перейти на
  `claude-sonnet-4-6` (через `ANTHROPIC_MODEL`) — почти уберёт; цена ~×3
  (~$1–2 / 100 ответов вместо ~$0.3–0.5).
- **Интент `biz`** не имеет дока в корпусе — агент отвечает по ближайшим
  (медицинским) докам корректно, но можно добавить отдельный документ для
  бизнес-виз/автономо.
- **«Цены» с €** в ответах — это лимиты покрытия из корпуса («возврат до
  1.500€»), не цена полиса. Корректно.

# UX-обзор приложений (web + admin) — что улучшить

Обзор по живому приложению через Playwright (web mobile/desktop + admin) с прогоном
E2E (12/12 зелёные). Фокус — удобство и «не перегруженный» интерфейс.

## 🔴 Дефекты, которые портят UX

- **D1 (ИСПРАВЛЕНО) Согласие показывало сырой HTML.** В контактной форме чекбокс
  выводил `…<a href="#privacy">Политика конфиденциальности</a>.` как текст — на
  самом ответственном шаге (отправка лида) это выглядело сломанным и било по доверию.
  Исправлено: чистый текст + отдельная кликабельная ссылка, открывающая модалку
  политики. (4 языка.)
- **D2 Reveal прячет ~половину лендинга.** Секции ниже сгиба анимируются появлением
  через `opacity-0` + IntersectionObserver. Риски: **SEO** (контент скрыт до
  срабатывания JS — важно, т.к. цель SEO/GEO), ощущение «пустой/медленной» страницы,
  поломка без JS. → Сделать контент **видимым по умолчанию**, анимацию оставить
  чистым усилением (не оставлять перманентный `opacity-0`); либо применять Reveal
  точечно. Лёгкая, но важная правка.

## 🟡 Перегруз и дублирование (главное по запросу «не перегружен»)

- **C1 Четыре одинаковых CTA «Подобрать страховку».** В навбаре, в hero, в секции
  «Подберём вашу страховку» (#quiz) и в плавающей кнопке (FAB). Секция #quiz теперь
  **только кнопка** (чат стал попапом) — это почти пустая секция-дубль. → **Убрать
  секцию #quiz** (или заменить полезным блоком: trust-сигналы / мини-FAQ). FAB +
  hero-CTA достаточно. Это убирает целую секцию и 2 лишних CTA — страница легче.
- **C2 Дубль «как это работает».** Карточка в hero («Как это устроено», 4 пункта) и
  отдельная секция «Как проходит подбор» (4 шага) рассказывают одно и то же. →
  Оставить одно (секция богаче), а в hero — короче/другой акцент (доверие, цифры).
- **C3 FAB громоздкий.** Всегда показывает полный текст «Подобрать страховку» и
  наезжает на карточки/контент. → Компактнее: иконка, расширяется по ховеру; или
  иконка-only на мобиле — меньше визуального шума.
- **C4 Повтор «Подобрать →» на каждой из 8 карточек** + те же CTA вокруг. → Вся
  карточка кликабельна; «Подобрать →» можно убрать, оставив заголовок+описание и
  понятный hover — меньше повторяющегося текста.

## 🟢 Удобство и поток

- **U1 Админ-таблица лидов: колонка «Срочность» обрезается** (горизонтальный скролл),
  колонок много. → Адаптивная таблица (карточки на узких экранах), приоритет колонок,
  + фильтр/смена статуса лида — заметно удобнее менеджеру.
- **U2 Контактная форма** (имя + мессенджер + контакт + согласие) — это разумный
  минимум для лида, ок. Прогресс-бар и «назад» хороши.
- **U3 Свободный ввод + чипсы** — реализовано хорошо; чипсы как подсказки, поле для
  своего вопроса. Можно добавить 2–3 «стартовых» чипса-вопроса на первом экране чата
  («Что нужно для визы?», «Сколько стоит?») для тех, кто не хочет проходить сценарий.

## Резюме (приоритет для «легче и удобнее»)
1. D2 — не прятать контент анимацией (SEO + восприятие). 
2. C1 — убрать дублирующую секцию #quiz (−1 секция, −2 CTA).
3. C2 — снять дубль «как это работает».
4. C3 — компактный FAB.
5. U1 — адаптивная admin-таблица + статус/фильтр лидов.

Эффект: минус одна секция и несколько повторяющихся CTA/текстов → интерфейс
ощутимо чище, путь к подбору остаётся в 1 клик (FAB/hero/карточки → чат).

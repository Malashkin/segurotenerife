# UX-аудит интерфейса Seguro Tenerife

Аудит проведён по живому приложению через Playwright (desktop 1366×900 и mobile
390×844, локаль ru-RU), скриншоты всех состояний: лендинг, hero, секции, FAB,
открытый чат, шаги подбора, форма, хендофф, admin-логин и дашборд.

Цель продукта: **посетитель получает удовольствие от простоты подбора страховки**
— не «как обычно» (формы, звонки, навязчивость), а спокойный разговор за 2 минуты.
Аудит оценивает интерфейс против этой цели.

## Сводка состояния
Фундамент сильный: чистая бренд-вёрстка, понятная иерархия, мультиязычность,
плавающий чат-бот, нейтральный честный тон («бесплатно», «без звонков»). Но есть
один критичный визуальный баг (уже исправлен) и набор трений в чат-флоу и на мобиле,
которые мешают цели «удовольствие от простоты».

## Находки (по приоритету)

### P0 — критично
- **F1 (ИСПРАВЛЕНО) Сырой HTML в заголовке hero.** Рендерилось буквально
  `Подбор <span class="hl">страховки</span> на Тенерифе…`. Причина: в словарь
  `hero_h1` положили HTML, хотя компонент `Hero` сам подсвечивает слово из
  `hero_h1_hl`. Исправлено: убраны теги во всех 4 локалях → заголовок чистый,
  слово «страховки» подсвечено бренд-цветом. **Это самый заметный дефект страницы.**

### P1 — высокий импакт на «простоту»
- **F2 Чат повторно спрашивает язык.** Посетитель уже выбрал язык на странице
  (или он определён по браузеру), а первый шаг чата — снова «На каком языке вам
  удобно общаться?». Лишнее трение на самом старте. В дашборде видно следствие:
  у лида, пришедшего из RU-интерфейса, `comm_lang = English` (авто-выбор первой
  опции). → Предзаполнять язык общения из языка страницы; шаг превратить в
  необязательное подтверждение или убрать.
- **F3 Нет индикатора прогресса в чате.** Пользователь не видит, сколько шагов
  осталось («2 из 6»). Неизвестность повышает отказы. → Тонкий прогресс-бар/счётчик
  в шапке чата.
- **F4 Мобильный чат — плавающая карточка, а не полноэкранный sheet.** На 390×844
  попап висит снизу карточкой; кнопки опций и форма уходят под обрез / требуют
  скролла в маленькой области. → На мобиле открывать как bottom-sheet почти на
  весь экран (как настоящие чат-боты).
- **F5 Нельзя поправить предыдущий ответ.** В сторе есть `back()`, но в UI нет
  кнопки «назад»/правки. Ошибся на шаге — только рестарт. → Кнопка «← изменить».

### P2 — польза/доверие
- **F6 Чат пока не отвечает на вопросы по страховкам.** Главная ценность («приятно
  и просто разобраться») недозакрыта: бот только собирает ответы и передаёт лид.
  База знаний ASISA уже собрана (`knowledge-base/`) — её надо подключить, чтобы бот
  отвечал на «что покрывает полис для ВНЖ?» до хендоффа.
- **F7 Хендофф-ссылки на плейсхолдерах.** Кнопки «Продолжить в WhatsApp/Telegram/
  Viber» используют дефолтные номера (env `VITE_*` не заданы). → Задать реальные
  контакты офиса перед продом.
- **F8 Дублирование «как это работает».** Карточка в hero и секция «Как проходит
  подбор» повторяют один контент. Не баг, но можно усилить дифференциацией.

### P3 — полировка
- **F9 Admin: таблица лидов горизонтально скроллится** (колонка «срочность»
  обрезана). → Адаптивная таблица / карточки на узких экранах.
- **F10 Фокус-менеджмент чата.** При открытии попапа фокус не переводится внутрь
  диалога, нет ловушки фокуса и закрытия по Esc. → Добавить для доступности.

## Что хорошо (не ломать)
- Чистая визуальная иерархия и бренд-палитра; читаемая типографика.
- Плавающий чат-бот + CTA-секция, открывающая его, — привычный паттерн.
- Честный нейтральный тон и trust-сигналы (бесплатно, без звонков, дисклеймер).
- Мультиязычность EN/ES/UA/RU с живым переключением (без перезагрузки).
- Лёгкий бандл (web ~124 КБ gz).
